Модификация агентной модели задачи о ферзях.

Задание: 
Доработать существующую агентно-ориентированную реализацию классической
задачи о расстановке N ферзей, добавив возможность поиска всех допустимых
конфигураций. Каждый ферзь (агент) должен быть экземпляром собственного,
отдельно определенного класса. Это позволит заложить основу для дальнейшей
индивидуализации поведения агентов. Оснастить программу графическим
интерфейсом для визуализации расположения фигур на шахматной доске.

Реализованные классы: 
1. QueenAgent - класс агента-ферзя
2. QueensSolver - класс для решения задачи о N ферзях с использованием агентов
3. ChessBoardWidget - виджет для отображения шахматной доски и ферзей
4. MainWindow - Главное окно приложения.

QueenAgent: 
Каждый ферзь представлен отдельным агентом, который управляет своим
положением на доске. Агенты взаимодействуют друг с другом через
сообщения для нахождения допустимых конфигураций.
Принцип работы:
- Каждый ферзь знает только своего соседа слева
- Ферзь проверяет, не атакует ли его сосед
- Если атакует - ферзь двигается вниз
- Если достиг дна - просит соседа подвинуться

QueensSolver отвечает за: 
- Создание цепочки агентов-ферзей
- Поиск всех допустимых решений
- Отправку найденных решений в главное окно через сигналы
- Управление потоком выполнения (асинхронный поиск)

ChessBoardWidget отвечает за: 
- Отрисовку шахматной доски с чередующимися клетками
- Отображение ферзей в виде стилизованных корон
- Навигацию по найденным решениям

MainWindow отвечает за: 
- Создание и расположение всех UI элементов
- Обработку действий пользователя (нажатие кнопок)
- Отображение прогресса поиска решений
- Управление анимацией решений
- Вывод информации в консоль
